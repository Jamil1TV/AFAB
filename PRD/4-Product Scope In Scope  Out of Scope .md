# 4. Product Scope (In Scope / Out of Scope)

## Purpose

This section defines the boundaries of AFAB Version 1 (MVP). It clearly identifies what will be developed in the initial release and what is intentionally postponed to future versions.

The objective is to maintain focus, reduce development complexity, and ensure that the MVP delivers a polished, reliable experience rather than an excessive number of incomplete features.

---

# 4.1 In Scope (Version 1)

The following features are included in the MVP and are considered essential for the first public release.

---

## User Authentication

* User Registration
* User Login
* Logout
* Email Verification
* Forgot Password
* Reset Password
* Remember Me
* User Profile Management
* Change Password
* Delete Account

---

## Dashboard

The dashboard serves as the central overview of the user's financial health.

It includes:

* Current Balance
* Monthly Income
* Monthly Expenses
* Savings Summary
* Budget Progress
* Recent Transactions
* Income vs Expense Charts
* Monthly Spending Charts
* Quick Actions

---

## Income Management

Users can:

* Create income records
* Edit income records
* Delete income records
* Organize income by category
* Create recurring income
* Search income
* Filter income
* Paginate records

---

## Expense Management

Users can:

* Create expenses
* Edit expenses
* Delete expenses
* Upload receipt images
* Categorize expenses
* Create recurring expenses
* Search expenses
* Filter expenses
* Paginate records

---

## Budget Management

Users can:

* Create monthly budgets
* Create category budgets
* Track budget usage
* Receive budget warnings
* View budget progress

---

## Savings Goals

Users can:

* Create savings goals

# 4.2 Out of Scope (Version 1)

The following features are intentionally excluded from the MVP.

They remain part of the long-term roadmap but will not be developed during Version 1.

---

## Artificial Intelligence

Excluded:

* AI Chat Assistant
* Financial Recommendations
* Budget Prediction
* Smart Expense Categorization
* AI Spending Insights
* AI Financial Health Score
* Natural Language Queries

---

## External Integrations

Not included:

* Bank APIs
* Open Banking
* Credit Card Synchronization
* Payment Gateways
* Accounting Software Integration

---

## Mobile Applications

Not included:

* Android Application
* iOS Application

The web application will be fully responsive.

---

## Multi-Business Support

Each user will manage a single business in Version 1.

Support for multiple businesses under one account will be introduced in future releases.

---

## Team Collaboration

Not included:

* Multiple Users
* Employee Accounts
* Account Roles
* Shared Workspaces

Version 1 is designed for a single business owner.

---

## Advanced Accounting

The following are intentionally excluded:

* Double-entry accounting
* Journal entries
* Tax calculations
* Payroll
* Inventory management
* Invoice generation
* Purchase orders

AFAB is a financial management platform, not a complete ERP or accounting system.

---

## Advanced Analytics

Excluded:

* Predictive forecasting
* Machine learning insights
* Business benchmarking
* Market comparison

---

# 4.3 Design Principles

Every feature included in Version 1 must satisfy the following principles:

* Solves a real user problem.
* Aligns with the MVP objectives.
* Can be implemented with high quality.
* Fits within the planned development timeline.
* Supports future scalability.

If a proposed feature does not satisfy these principles, it should be postponed to a future version.

---

# 4.4 Scope Control Policy

Any new feature request must answer the following questions before being accepted into the MVP:

1. Does it solve a core problem for small businesses?
2. Does it align with AFAB's product vision?
3. Can it be completed without delaying the MVP?
4. Does it introduce unnecessary complexity?
5. Can it be postponed without reducing the value of Version 1?

If the answer to Questions 3, 4, or 5 indicates increased risk or unnecessary complexity, the feature should be moved to the product roadmap instead of the MVP.
