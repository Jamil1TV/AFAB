# AFAB Frontend Coding Standards

## Goals

-   Strict TypeScript
-   Feature-first architecture
-   Server Components by default
-   Reusable UI

## Naming

-   Components: PascalCase
-   Hooks: useXxx
-   Stores: useXxxStore
-   Types: XxxProps / XxxResponse
-   Constants: UPPER_SNAKE_CASE

## React Rules

-   Keep components \<300 lines
-   Extract logic into hooks
-   Avoid prop drilling
-   Prefer composition over inheritance

## Next.js

-   App Router
-   Route Groups
-   Server Actions only when appropriate
-   Metadata API for SEO

## Imports

1.  React/Next
2.  Third-party
3.  Internal aliases
4.  Relative

## Error Handling

-   Error boundaries
-   Typed API errors
-   Toast + inline validation
